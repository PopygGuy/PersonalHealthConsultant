# PersonalHealthConsultant
Развертывание мобильного приложения «Персональный консультант по здоровью».

Текущая реализация выполнена на Flutter (Dart) и включает роли администратора, преподавателя и студента, журнал оценок нормативов, а также модуль здоровья с персональными рекомендациями.

## Последние изменения

Добавлен отдельный шагомер для студентов с сохранением данных между перезапусками приложения.

- Добавлен новый экран `lib/screens/student/step_tracker_screen.dart`.
- В `StudentDashboard` добавлена отдельная вкладка **Шагомер**.
- Реализована персональная статистика шагов по пользователю (по `user.id`) через `SharedPreferences`:
  - шаги за текущий день;
  - суммарные шаги;
  - история за последние дни.
- Добавлено запоминание сессии пользователя:
  - новый сервис `lib/services/session_service.dart`;
  - автологин при старте приложения;
  - очистка сессии при выходе из профиля.
- Обновлены платформенные настройки для шагомера:
  - Android: `ACTIVITY_RECOGNITION` в `AndroidManifest.xml`;
  - iOS: `NSMotionUsageDescription` в `Info.plist`.
- Подключены зависимости:
  - `pedometer`;
  - `permission_handler`.

## Централизованная БД (рекомендуется)

В проекте должна использоваться **одна серверная БД**:

- канонический файл SQLite: `backend/phc.db`;
- логическая структура внутри одной БД: `users`, `faculties`, `groups`, `norms`, `grades`, `step_stats`.

### Почему это важно

- нет дублирования данных;
- нет рассинхронизации между ролями и справочниками;
- проще администрирование и сопровождение.

### Разовая централизация данных

Если ранее данные случайно расходились между `phc.db` (в корне) и `backend/phc.db`, выполните:

- `.\backend\.venv\Scripts\python.exe backend/scripts/centralize_db.py`

Скрипт переносит справочники факультетов/групп/нормативов в `backend/phc.db` и корректно перепривязывает группы к факультетам.
