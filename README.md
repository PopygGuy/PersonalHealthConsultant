# Централизованная система мониторинга физической подготовки и академической оценки студентов

Проект представляет собой клиент-серверную информационную систему для цифровизации процессов в образовательной среде:

- учёт и контроль оценок по нормативам;
- мониторинг физической активности студентов;
- централизованное управление пользователями, факультетами и группами.

Система ориентирована на поэтапное внедрение: от уровня факультета до уровня всего университета.

## Кратко для пользователя

Это единое приложение для трёх ролей:

- студент видит свои оценки и физическую активность;
- преподаватель выставляет и контролирует оценки;
- администратор управляет структурой (факультеты, группы, пользователи).

Главная идея: все данные хранятся централизованно, без дублирования и разрозненных таблиц.

## Цель проекта

Создать единую цифровую платформу, которая:

- упрощает контроль физической подготовки и учебных результатов;
- повышает прозрачность выставления оценок;
- исключает дублирование данных;
- обеспечивает масштабируемость и управляемость.

## Архитектура

Система построена по клиент-серверной архитектуре:

`Flutter (Android/Web) -> REST API (FastAPI/Python) -> Централизованная БД (SQLite, с готовностью к PostgreSQL)`

Почему выбран такой подход:

- нет прямого доступа клиента к БД;
- единая точка контроля и валидации;
- удобное масштабирование;
- поддержка разных устройств с единым UX.

## Роли и доступ

Реализована ролевая модель:

- **Студент**
  - просмотр собственных оценок;
  - просмотр статистики активности;
  - персональные функции профиля и шагомера.
- **Преподаватель**
  - выставление и просмотр оценок;
  - фильтрация студентов по факультетам и группам;
  - мониторинг результатов.
- **Администратор**
  - управление пользователями;
  - управление факультетами, группами, нормативами;
  - сервисные операции восстановления и нормализации справочников.

## Основной функционал

1. **Учёт академических оценок**
   - выставление и обновление оценок по нормативам;
   - хранение истории изменений;
   - удобная фильтрация и поиск.
2. **Мониторинг физической активности**
   - шагомер для студентов;
   - расчёт дистанции и накопительной статистики;
   - персонализация на основе параметров пользователя.
3. **Централизованные справочники**
   - единая БД с факультетами, группами, нормативами;
   - устранение дублей и рассинхронизации.
4. **Адаптивный UI/UX**
   - единый стиль на мобильных и web-экранах;
   - глобальное переключение темы (светлая/тёмная/системная);
   - персонализированные аватары пользователей.

## Технологии и обоснование выбора

- **Flutter (Dart)**
  - единая кодовая база для Android и Web;
  - высокая скорость разработки;
  - адаптивный интерфейс.
- **Python + FastAPI**
  - быстрая реализация REST API;
  - читаемость и удобство сопровождения;
  - понятная интеграция с ORM.
- **SQLAlchemy (ORM)**
  - абстракция от СУБД;
  - снижение количества SQL-ошибок;
  - упрощение миграции SQLite -> PostgreSQL.
- **SQLite (этап пилота)**
  - лёгкий запуск без отдельного сервера;
  - подходит для учебного/факультетского старта.
- **PostgreSQL (этап масштабирования)**
  - целевая СУБД для крупного внедрения;
  - готовность к росту нагрузки и данных.

## Централизованная БД (важно)

В проекте используется единая серверная БД:

- канонический файл SQLite: `backend/phc.db`;
- сущности: `users`, `faculties`, `groups`, `norms`, `grades`, `step_stats`.

Почему это критично:

- нет дублирования и конфликтов;
- единая версия данных для всех ролей;
- простое резервное копирование и поддержка.

### Сервисные скрипты

- Централизация данных из legacy-файла:
  - `.\backend\.venv\Scripts\python.exe backend/scripts/centralize_db.py`
- Удаление дублей факультетов и перепривязка связей:
  - `.\backend\.venv\Scripts\python.exe backend/scripts/dedupe_faculties.py`
- Заполнение/дополнение справочников:
  - `.\backend\.venv\Scripts\python.exe backend/scripts/seed_initial_catalog.py`
- Создание/обновление администратора `root/root`:
  - `.\backend\.venv\Scripts\python.exe backend/scripts/bootstrap_admin.py`

## Быстрый старт (локальный запуск)

### 1) Backend (FastAPI)

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe scripts\bootstrap_admin.py
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

После запуска API доступен по адресу: `http://127.0.0.1:8000`

### 2) Flutter-клиент

```powershell
cd ..
flutter pub get
flutter run
```

Тестовый вход:

- администратор: `root / root`
- преподаватель: `teacher / 123` (если создан)
